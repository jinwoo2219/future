# FUTURE Fun Validation Checklist

## Purpose

Use this checklist before focusing on full release polish.
This file is for answering one question:

**"Is the current version actually fun to play again?"**

---

## 1. Early Game Feel

- [ ] The first 2 turns are understandable without outside explanation
- [ ] The player can identify what each starting card is roughly for
- [ ] The first stage does not feel solved or automatic
- [ ] The player has at least one meaningful choice in the first stage
- [ ] Early enemies create pressure without feeling unfair

## 2. Choice Quality

- [ ] Stage choices feel meaningfully different
- [ ] Choosing enemies changes later fights in a noticeable way
- [ ] Card rewards create real hesitation instead of obvious picks
- [ ] At least 2 viable strategies exist in a typical run
- [ ] The player can make a risky choice on purpose and feel the result later

## 3. Repetition Check

- [ ] 3 turns in a row do not feel identical in normal play
- [ ] The same card pattern does not solve most early fights
- [ ] The same enemy pattern does not dominate every stage
- [ ] The same install or control line does not trivialize all encounters
- [ ] Different runs feel meaningfully different by stage 3

## 4. Power Growth

- [ ] The player feels stronger over time during a run
- [ ] Card rewards actually change how turns are played
- [ ] Relics meaningfully alter decisions or power
- [ ] New enemies increase complexity without removing player agency
- [ ] Boss fights feel like a test of the run that was built

## 5. Frustration Check

- [ ] Defeats usually feel understandable
- [ ] The player can identify at least one mistake after losing
- [ ] Losses do not feel mostly caused by unreadable rules
- [ ] UI does not hide important state during combat
- [ ] No single enemy or card interaction feels unfair enough to stop another run

## 6. Card Balance Check

- [ ] No starter card feels useless
- [ ] No single common card feels mandatory
- [ ] Rare and epic cards feel exciting without instantly breaking the run
- [ ] Install, utility, control, and damage all have meaningful use cases
- [ ] Cooldown, action cost, and effect strength feel internally consistent

## 7. Stage 1 Validation

- [ ] Stage 1 is fun on repeated play
- [ ] Stage 1 teaches the game naturally
- [ ] Stage 1 does not drag or feel too short
- [ ] Stage 1 boss feels like a payoff, not just a rules exception
- [ ] After clearing Stage 1, the player wants to continue

## 8. Current Priority Test

- [ ] After one full test run, I want to immediately do another run
- [ ] I can name at least 3 cards or interactions that felt genuinely fun
- [ ] I can name at least 3 weak or boring parts that need fixing
- [ ] The next balance change I want to make is obvious
- [ ] The current build is worth showing to another person
